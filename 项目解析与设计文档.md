## 三、数据的两种获取方式（你提到的核心问题）

### 3.1 方式一：本地部署商城，调用 ETL 接口

**原理**：商城源码里 `server/src/server.js` 已经内置了 `/api/etl/*` 只读接口，你只需在本地启动商城，就能通过这些接口获取数据。

**步骤**：
```bash
# 1. 安装商城依赖并生成数据
npm install --prefix server
cd server && node src/seed.js && cd ..

# 2. 启动商城 API
npm start --prefix server

# 3. 然后就能访问 ETL 接口了：
# http://localhost:38173/api/etl/tables      ← 列出所有可用表
# http://localhost:38173/api/etl/schema/dim_user  ← 查看表结构
# http://localhost:38173/api/etl/query/fact_order?limit=100  ← 分页查询
# http://localhost:38173/api/etl/export/dim_user?format=csv   ← 导出 CSV
# http://localhost:38173/api/etl/quality      ← 数据质量报告
# http://localhost:38173/api/etl/metrics       ← 核心经营指标
```

**老师在讲义中提到的方式**：
```python
from course_utils.data_loader import API_BASE, get_metrics, get_quality_report, get_table_catalog
```

这是老师封装好的 Python 库，内部就是调 ETL 接口。但老师说他的内网你们访问不到，所以你们需要自己在本地启动商城。

### 3.2 方式二：直接用 CSV 构建本地分析环境

**原理**：`exports/` 目录下的 CSV 文件和 SQLite 数据库里的分析视图完全对应。你可以用 pandas 直接读取 CSV，不依赖商城是否启动。

```python
import pandas as pd

# 直接读取 CSV，不依赖网络和商城
users = pd.read_csv("exports/dim_user.csv")
orders = pd.read_csv("exports/fact_order.csv")
order_items = pd.read_csv("exports/fact_order_item.csv")
traffic = pd.read_csv("exports/fact_traffic.csv")
daily = pd.read_csv("exports/daily_business_summary.csv")
```

### 3.3 两种方式对比

| 对比维度 | ETL 接口 | CSV 直接读 |
|----------|---------|-----------|
| 依赖 | 需要先启动商城 `npm start --prefix server` | 不需要任何服务 |
| 灵活性 | 支持 SQL 过滤、排序、分页、动态查询 | 读到内存后自己用 pandas 操作 |
| 实时性 | 数据随商城运行可更新 | 静态快照 |
| 适合谁 | 做仪表盘时，动态刷新数据 | 做探索分析、跑 notebook 时 |
| **建议** | **仪表盘后端用 ETL 接口** | **探索分析阶段用 CSV** |

**最佳实践组合**：开发阶段用 CSV 快速探索和分析，模型成型后，仪表盘后端通过 ETL 接口或 SQLite 直连获取数据。

## 六、维度切片设计

"切片"就是从一个数据立方体的不同角度切开来查看数据。这个项目里有哪些维度可以做切片？

### 6.1 时间维度（最基础的切片轴）

| 切片粒度 | 用途 | 数据来源 |
|----------|------|---------|
| **按天** | 看每日波动，找异常日 | `daily_business_summary.summary_date` |
| **按周** | 看周趋势，发现周末效应 | 从 `summary_date` 聚合 |
| **按月** | 看月度趋势，做环比/同比（老师重点强调） | 从 `summary_date` 聚合（排除不完整月份！） |
| **按年** | 年度对比 | 从 `summary_date` 提取年份 |
| **工作日 vs 周末** | 看消费时段差异 | `dim_date.is_weekend` |
| **最近 7/30/90 天** | 模型特征的时间窗口 | 用于特征工程 |

### 6.2 渠道维度（回答"用户从哪来"）

| 切片值 | 含义 | 典型分析问题 |
|--------|------|-------------|
| `organic` | 自然流量（直接访问、自然搜索） | GMV 贡献最高（约 26.81%），需要重点诊断 |
| `search` | 搜索广告/搜索入口 | 付费渠道，需要计算 ROAS |
| `social` | 社交媒体 | 拉新效果好？ROI 高？ |
| `email` | 邮件触达 | 复购召回渠道 |
| `affiliate` | 联盟渠道 | 佣金成本 vs 收入 |
| `live` | 直播 | 虽然体量小，但转化率可能高（1.35x） |

**示例**：看"本月各渠道 GMV 贡献"→ 渠道维度切片。哪条渠道转化率在下降？→ 渠道 × 漏斗维度切片。

### 6.3 类目维度（回答"什么品类卖得好"）

12 个一级类目，从 `dim_product.category_name` 获取：
- 美妆个护、家居生活、食品饮料、数码电子
- 运动户外、母婴用品、服饰鞋包、图书文娱
- 汽车用品、珠宝饰品、医疗保健、宠物生活

**示例**：看"哪些类目 GMV 高但毛利低"→ 类目 × GMV × 毛利切片。

### 6.4 用户维度（回答"谁在买"）

### 6.5 商品维度（回答"什么商品值得关注"）

### 8.1 数据读取层（data_access.py）应支持三种模式

```python
# 模式 1：CSV 直接读取（离线探索用）
df = pd.read_csv("exports/fact_order.csv")

# 模式 2：SQLite 直连（仪表盘生产用）
import sqlite3
conn = sqlite3.connect("file:server/data/eshop.sqlite?mode=ro", uri=True)

# 模式 3：ETL HTTP API（远程/容器化部署用）
import requests
```

## 十一、关键注意事项（老师课上强调过的）

1. **不完整月份不能做环比**：如果今天是 3 月 10 日，不要拿 3 月跟 2 月整月比。代码里要排除当前不完整的月份。

2. **数据泄漏是死罪**：预测未来复购时，特征只能用预测时间点之前的数据。用了未来信息，模型离线好看，上线就会崩。

3. **每个指标都要说清楚口径**：GMV 用的是 `SUM(paid_amount) WHERE status IN ('paid','completed')`，不是随便一个数。这也是答辩时老师必问的。

4. **GMV ≠ 利润**：GMV 是交易规模，毛利率反映赚钱能力。低毛利高 GMV 的增长质量可能很差。

5. **不要做全量粗放发券**：发券要有目标人群和对照组。不设对照组就分不清是券的效果还是自然复购。

6. **维度表和事实表的区别**：维度表描述对象（用户是谁、商品是什么），事实表记录事件（订单、浏览、退款）。
