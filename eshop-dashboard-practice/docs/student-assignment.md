# 学生作业说明

## 作业目标

基于教师提供的在线商城与 `course_dataset_v2` 数据，完成经营管理与营销决策场景下的数据挖掘、可视化和后台运营支持系统。

## 不要求完成

- 不要求开发商城顾客端主体流程；
- 不要求实现购物车、下单、支付等核心交易闭环；
- 不要求搭建完整 ETL 调度系统或爬取商业平台商品数据。

## 必做内容

1. 数据挖掘与可视化
   - 至少覆盖 4 个主题：转化、复购、RFM、品类贡献、活动效果、渠道 ROI、履约售后、库存动销等；
   - 提供 notebook 或等价可复现代码；
   - 每个指标必须说明表、字段、过滤条件和粒度。
   - 至少完成一个模型型任务：复购预测、流失预警、券敏感度、需求预测、退款风险中的任意一个。

2. 后台运营管理与营销支持
   - 至少包括商品/订单/用户中的两个管理模块；
   - 至少包括一个营销模块：优惠券、活动、用户标签或活动报表；
   - 至少 3 个可刷新报表或导出功能；
   - 所有运营动作应写入 `admin_action_logs` 或等价审计表。

3. 决策说明
   - 用数据给出 3 条以上经营建议；
   - 说明建议对应的数据证据、适用范围和风险。
   - 至少一条建议应落到预算、优惠、选品、库存或履约策略之一。

## 数据读写边界

- 可读：所有 `dim_*`、`fact_*`、`daily_business_summary` 和公开 API。
- 可写：营销配置、运营备注、用户标签、后台审计、新增活动测试数据。
- 不建议写：历史订单金额、订单行价格、支付成功流水。

## 评分建议

- 数据挖掘与可视化：45%
- 后台管理与营销决策支持：40%
- 指标口径、复现与答辩：15%
