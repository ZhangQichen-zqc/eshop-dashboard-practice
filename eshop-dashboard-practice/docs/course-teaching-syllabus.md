# 数据挖掘与商业决策课程教案

## 课程定位

本课程基于 `course_dataset_v2` 电商经营数据集开展。商城前台只是数据源演示环境，课程重点是让学生完整经历：

1. 从商城源系统理解业务与数据结构；
2. 进行 ETL、数据清洗、指标建模和特征工程；
3. 应用分类、聚类、关联规则、时间序列、营销效果评估等数据挖掘方法；
4. 将模型结果落到经营管理与营销决策支持系统中。

## 课程数据资产

核心数据包括：

- 用户：注册渠道、地区、会员等级、生命周期分群。
- 商品：类目、品牌、SKU、价格、成本、供应商、库存流水。
- 流量：未转化会话、浏览、搜索、加购、结算、支付成功。
- 交易：订单、订单行、支付、优惠、活动、退款。
- 营销：活动、优惠券、广告曝光/点击/转化/消耗。
- 履约与体验：物流时效、延迟、评论评分、评论标签。

主要分析视图：

- `dim_date`

## 总体教学目标

学生完成课程后应能：

- 解释电商业务表、分析表和指标口径之间的映射关系。
- 独立完成数据清洗、缺失处理、异常检测、派生特征和宽表构建。
- 面向经营问题选择合适的数据挖掘方法，而不是机械套模型。
- 构建可复现的数据分析 notebook 或脚本。
- 将分析结果实现为后台运营管理与决策支持功能。
- 用数据给出经营建议，并说明证据、限制和风险。

## 与原课程 1-7 章的覆盖关系

本教案以 `Data_Mining_Management_Decision_Course_20260228` 的第 1-7 章为基础，并用统一的电商经营数据集贯穿全部教学内容。覆盖关系如下：

| 原课程章节 | 原有核心内容 | 本项目中的承接方式 |
|---|---|---|
| 第 1 章：数据挖掘概述 | 数据挖掘定义、发展、管理决策价值、CRISP-DM、Python/Jupyter 工具生态 | 第 1 周引入 CRISP-DM，并要求学生用 Jupyter/Notebook 或脚本完成可复现分析 |
| 第 2 章：数据准备与探索 | 结构化/半结构化/非结构化数据，多源读取，数据质量评估，EDA，可视化 | 第 2-3 周扩展为 SQLite/CSV/API 数据接入、质量报告、经营驾驶舱 |
| 第 3 章：数据预处理 | 缺失值、异常值、重复值、标准化、归一化、编码、特征选择、降维、类别不平衡 | 第 2、5、6 周要求学生构建清洗流水线和建模宽表，处理类别变量和不平衡样本 |
| 第 4 章：分类算法 | 业务分类问题、逻辑回归、决策树、随机森林、梯度提升、模型评估、A/B 测试 | 第 5-6 周落到复购、流失、券敏感、退款风险等电商分类任务 |
| 第 5 章：聚类分析 | K-Means、层次聚类、客户细分、产品细分、轮廓系数、业务解释 | 第 7 周落到用户分群和商品分群，并要求生成运营策略 |
| 第 6 章：关联规则 | Apriori、FP-growth、购物篮、支持度、置信度、提升度、交叉销售 | 第 8 周落到 SKU/类目组合、凑单推荐、捆绑销售 |
| 第 7 章：时间序列 | 趋势、季节性、平稳性、ADF/KPSS、ACF/PACF、ARIMA、销售预测 | 第 9 周落到 GMV、品类销量、SKU 需求预测与补货决策 |

扩展内容包括但不限于：营销增量评估、履约售后治理、库存决策、文本/评论分析、异常检测、推荐系统和综合决策支持系统。

## 14 周教学大纲

### 第 1 周：业务理解与数据源认知

教学内容：

- 数据挖掘定义、核心特征，以及与统计学、机器学习、大数据的区别。
- CRISP-DM 六阶段：业务理解、数据理解、数据准备、建模、评估、部署。
- Python 数据挖掘工具生态：Jupyter Notebook、pandas、NumPy、scikit-learn、matplotlib/seaborn。
- 电商平台基本业务流程：流量、商品、购物车、订单、支付、履约、退款、评价。
- OLTP 源系统与分析数据集的区别。
- 主键、外键、粒度、时间口径、事实表与维度表。

课堂任务：

- 配置 Python/Jupyter 环境，连接 SQLite 或读取导出的 CSV。
- 阅读 `docs/source-to-dataset-mapping.md`。
- 画出 `users`、`sku`、`orders`、`order_items`、`page_events` 到分析视图的映射。
- 按 CRISP-DM 写出电商经营分析项目计划。

决策问题：

- 一个运营经理每天需要看哪些指标？
- 商品、用户、渠道、活动各自对经营有什么影响？
- 如何把“提升销售额”转化为可建模的数据挖掘目标？

### 第 2 周：ETL 与数据清洗

教学内容：

- ODS、DWD、DWS、ADS 的简化分层。
- 数据来源与类型：SQLite 数据库、CSV 导出、API、JSON；结构化、半结构化、非结构化数据。
- 数据质量维度：完整性、准确性、一致性、时效性、唯一性。
- 数据清洗：类型转换、重复记录、缺失值、异常值、状态过滤。
- 指标口径：GMV、净销售额、订单数、客单价、毛利、退款率、转化率。

课堂任务：

- 使用 pandas 分别从 SQLite 与 CSV 读取同一张分析表，并核对行数和字段类型。
- 生成数据质量报告：缺失率、重复主键、值域异常、时间范围、状态分布。
- 基于 `fact_order`、`fact_order_item`、`fact_refund` 构建经营日表。
- 校验订单金额、订单行金额、优惠分摊、退款金额是否满足基本约束。

决策问题：

- 哪些订单应该纳入 GMV？
- 退款应该何时冲减销售额？
- 同一个指标为什么会出现多个口径？
- 数据质量问题会如何误导经营决策？

### 第 3 周：探索性分析与经营驾驶舱

教学内容：

- 缺失值处理：删除、统计量填充、插值、KNN/模型填充的适用条件。
- 异常值处理：IQR、Z-score、Isolation Forest、LOF。
- 特征缩放：标准化、归一化、鲁棒缩放。
- 特征编码：One-hot、Ordinal、Target Encoding 的风险。
- 特征选择与降维：相关性、方差过滤、PCA，及其业务解释限制。
- 单变量、双变量、多维切片。
- 趋势、结构、对比、贡献度、帕累托分析。
- 可视化误区：双轴误导、截断坐标、口径不一致。

课堂任务：

- 构建可复用预处理流水线：清洗、编码、缩放、特征选择。
- 按日期、渠道、类目、会员等级构建经营驾驶舱。
- 输出 GMV、订单数、客单价、毛利、退款率、转化率。
- 比较预处理前后对后续模型特征分布的影响。

决策问题：

- 当前增长主要来自用户增长、客单提升还是转化提升？
- 哪些品类贡献高但毛利低？
- 哪些异常点应该保留为真实经营异常，而不是简单删除？

### 第 4 周：用户分析与 RFM

教学内容：

- Recency、Frequency、Monetary。
- cohort 留存与复购。
- 用户生命周期：新客、活跃、高价值、沉默、流失风险。

课堂任务：

- 构建用户 RFM 宽表。
- 输出用户分层策略：高价值、潜力、价格敏感、待唤醒。

决策问题：

- 哪些用户应发大额券？
- 哪些用户适合会员权益而不是折扣？

### 第 5 周：分类模型一，复购与流失预测

教学内容：

- 分类问题定义：标签窗口、观察窗口、预测窗口。
- 特征工程：购买频次、最近购买、品类偏好、券使用、访问行为、退款体验。
- 分类算法：逻辑回归、决策树、随机森林、梯度提升。
- 可选进阶算法：支持向量机、神经网络、XGBoost/LightGBM。
- 训练集/测试集时间切分。
- 准确率、召回率、AUC、混淆矩阵、业务成本。
- 模型解释：特征重要性、局部解释、业务可解释性。

课堂任务：

- 对比至少 3 个分类算法，并给出算法选择依据。
- 构建 30/60/90 天复购预测或流失预警模型。
- 输出高风险用户名单和运营动作建议。

决策问题：

- 召回率和精确率哪个更重要？
- 如果唤醒券成本为 20 元，应如何确定触达阈值？
- 如何避免时间穿越和数据泄漏？

### 第 6 周：分类模型二，券敏感与退款风险

教学内容：

- 券敏感用户识别。
- 退款风险识别。
- 类别不平衡、阈值调优、特征重要性解释。
- 业务验证：A/B 测试、对照组、触达成本与收益模拟。
- 与 AI 助手协作：算法选择、特征解释、实验设计、报告润色。

课堂任务：

- 基于 `fact_coupon_use` 训练券核销倾向模型。
- 或基于 `fact_refund`、`fact_fulfillment`、`fact_product_review` 训练退款风险模型。
- 设计一个模型上线后的 A/B 测试方案。

决策问题：

- 哪些用户不需要优惠也会购买？
- 哪些订单应提前客服介入？
- 模型分数如何转化为运营名单和预算？

### 第 7 周：聚类分析与用户/商品分群

教学内容：

- K-Means、层次聚类、DBSCAN 的适用条件。
- 标准化、距离度量、肘部法、轮廓系数。
- 聚类评估：轮廓系数、Calinski-Harabasz、Davies-Bouldin。
- 分群命名与业务解释。
- 聚类结果业务验证：专家评审、A/B 测试、策略稳定性。

课堂任务：

- 用户聚类：购买力、活跃度、品类偏好、券敏感、退款体验。
- 商品聚类：价格、毛利、销量、退款率、评分、库存周转。
- 输出每个簇的画像、命名、规模、价值和运营策略。

决策问题：

- 不同用户群应该采用什么触达策略？
- 商品应该分为引流款、利润款、长尾款、风险款还是清仓款？
- 聚类结果如何证明“有业务价值”而不仅是数学分组？

### 第 8 周：关联规则与购物篮分析

教学内容：

- 支持度、置信度、提升度。
- 订单行数据转换为购物篮。
- Apriori 与 FP-growth。
- 频繁项集、关联规则剪枝、规则冗余和搭配推荐。
- 规则业务评估：销量规模、毛利、库存约束、页面推荐位可用性。

课堂任务：

- 基于 `fact_order_item` 挖掘类目或 SKU 之间的关联。
- 比较按 SKU、类目、价格带构造购物篮时规则结果的差异。
- 设计商品捆绑、凑单、交叉销售推荐。

决策问题：

- 哪些商品适合组合销售？
- 哪些商品适合作为满减凑单推荐？
- 高提升度但低支持度的规则是否值得上线？

### 第 9 周：时间序列分析与需求预测

教学内容：

- 趋势、季节性、节假日/活动冲击。
- 移动平均、指数平滑、ARIMA/Prophet 思路。
- 平稳性检验：ADF、KPSS。
- ACF/PACF 与 ARIMA 参数识别。
- 时间序列分解：趋势、季节性、随机项。
- 训练集/验证集时间切分。
- 预测误差：MAE、MAPE、RMSE。

课堂任务：

- 预测日 GMV、品类销量或 SKU 需求。
- 识别活动期异常峰值和节假日季节性。
- 对至少一个序列完成平稳性检验、差分、ACF/PACF 分析和模型诊断。

决策问题：

- 未来两周应该补哪些 SKU？
- 哪些销售增长只是活动透支？
- 销售预测误差会如何影响库存成本和缺货风险？

### 第 10 周：营销效果评估与活动增量

教学内容：

- ROI、ROAS、CAC、转化成本。
- 对照组、准实验、活动前后对比。
- uplift 思想：谁因活动而购买。

课堂任务：

- 基于 `dim_campaign`、`fact_ads_spend`、`fact_order` 评估活动。
- 对有 `has_control_group` 的活动做增量分析。

决策问题：

- 哪些活动值得继续投放？
- 预算应该从哪个渠道迁移到哪个渠道？
- 优惠券是带来新增还是补贴原本会买的人？

### 第 11 周：履约、售后与体验分析

教学内容：

- 延迟率、退款率、评分、评论标签。
- 履约体验对复购的影响。
- 供应商、承运商、品类的风险比较。

课堂任务：

- 分析配送延迟是否提高退款率。
- 找出高退款、高差评、高毛利损失的商品或供应商。

决策问题：

- 哪些供应商需要治理？
- 哪些品类应调整页面说明或质检标准？

### 第 12 周：库存与供应链决策

教学内容：

- 动销率、库存周转、滞销、缺货风险。
- ABC 分类。
- 需求预测与补货建议。

课堂任务：

- 基于 `fact_inventory_movement` 和 `fact_order_item` 计算 SKU 动销。
- 给出补货、清仓、停售建议。

决策问题：

- 哪些 SKU 应提高库存？
- 哪些 SKU 应降价清仓？
- 高毛利但低动销商品如何处理？

### 第 13 周：后台运营决策支持系统开发

教学内容：

- 从 notebook 到运营系统。
- 指标卡、筛选器、钻取、导出、模型结果落库。
- 操作审计与指标口径说明。

课堂任务：

- 学生实现后台模块：经营驾驶舱、用户分群、活动复盘、商品策略、库存预警等。
- 所有运营动作写入 `admin_action_logs` 或等价审计表。

决策问题：

- 分析结果如何转化为运营动作？
- 模型推荐是否需要人工审核？

### 第 14 周：综合答辩

教学内容：

- 数据证据链：业务问题、数据口径、模型方法、结果解释、经营建议。
- 模型局限性与伦理：合成数据、隐私、过度优惠、偏见。

课堂任务：

- 展示后台系统、Dashboard、模型结果和经营建议。
- 提交可复现代码、数据字典引用和决策报告。

决策问题：

- 如果只能执行三项经营动作，应选哪三项，为什么？
