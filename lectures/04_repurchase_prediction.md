# 第 4 章 复购预测与触达名单：把模型分数变成发券决策

## 1. 本章导读：不是预测谁会买，而是决定钱花给谁

本章进入第一个预测模型：复购预测。

很多同学会以为预测模型的目标是“猜准谁会买”。这个理解只对了一半。在商业中，复购预测真正服务的是资源分配。

运营预算有限，不能给所有用户都发券。发给太多人，会浪费补贴；发给太少，活动规模不够；发给本来就会买的人，可能只是让他少付钱；发给完全不会买的人，又没有效果。

所以本章的核心问题是：

```text
在预算有限的情况下，应该优先触达哪些用户？
```

模型的作用是给用户排序。运营的作用是根据排序、成本和收益决定触达策略。

## 2. 本章学习目标

学完本章后，学生应该能够：

1. 解释什么是复购。
2. 定义观察窗口和预测窗口。
3. 区分特征和标签。
4. 理解分类模型在商业中的作用。
5. 用白话解释随机森林。
6. 理解 AUC、Precision、Recall。
7. 理解阈值、Top N 触达和 ROI。
8. 写出一份发券名单的商业建议。

## 3. 商业故事：50 万发券预算怎么用

运营总监说：

> 下个月大促，我们有 50 万发券预算。问题是，发给谁？

如果给所有用户发券，预算很快用完，而且很多用户可能不会买。如果只给老用户发券，又可能错过一些正在犹豫的新用户。

数据分析师需要做的是：

1. 根据历史行为估计用户未来复购可能性。
2. 按可能性排序。
3. 选择一批最值得触达的用户。
4. 估算触达成本和预期收益。

## 4. 商业概念 1：什么是复购

复购就是用户再次购买。

但在数据分析中，复购必须有明确时间范围。

错误说法：

```text
这个用户以后会不会复购？
```

这个说法太模糊。“以后”是 7 天、30 天、60 天，还是一年？

正确说法：

```text
在未来 60 天内，这个用户是否再次购买？
```

本章定义：

- 观察窗口：2026-01-01 之前的历史数据。
- 预测窗口：2026-01-01 之后 60 天。
- 标签：预测窗口内是否购买。

如果用户在未来 60 天购买，标签为 1；否则为 0。

## 5. 数据挖掘概念 1：分类模型

分类模型用于预测一个对象属于哪一类。

本章是二分类问题：

```text
1 = 会复购
0 = 不会复购
```

分类模型不会真正知道未来，它只是根据历史数据学习规律。

例如它可能发现：

- 最近买过的人更容易复购。
- 订单数多的人更容易复购。
- 近期浏览多的人更容易复购。
- 很久没买的人复购概率较低。

模型把这些规律组合起来，给每个用户一个复购概率。

## 6. 数据挖掘概念 2：随机森林是什么

随机森林可以用一个生活比喻理解。

假设你要判断一个用户会不会复购，你请 100 个经验丰富的运营同事投票。每个人根据不同线索判断：有人看最近购买，有人看消费金额，有人看浏览行为，有人看用券情况。最后综合大家的意见，得到一个概率。

随机森林就是很多棵决策树组成的“投票团队”。

每棵树都像一套判断规则：

```text
如果最近 30 天买过，并且历史消费高，则复购可能高。
如果很久没买，并且近期没有浏览，则复购可能低。
```

单棵树容易偏见很重，但很多棵树一起投票，通常更稳定。

## 7. 数据概念：观察窗口、预测窗口和数据泄漏

时间窗口是复购预测中最重要的概念之一。

我们站在 2026-01-01 这个时间点做预测。

```text
历史数据                         未来结果
--------------------|-------------------------
用于构造特征          用于判断是否复购
                  2026-01-01
```

历史数据可以用来构造特征，例如：

- 过去买过几次
- 过去花了多少钱
- 最近一次购买距今天数
- 最近是否浏览

未来 60 天只能用来构造标签，不能用来构造特征。

如果把未来订单数放进特征，模型就相当于偷看答案。这叫数据泄漏。

数据泄漏会让离线效果很好，但真实上线很差。

## 8. 模型指标 1：AUC

AUC 是分类模型常见指标。对零基础学生，可以这样理解：

> AUC 衡量模型把会复购用户排在不会复购用户前面的能力。

如果 AUC = 0.5，说明模型和随机猜差不多。

如果 AUC = 1，说明模型完美排序。

本章模型 AUC 约为 0.664，说明模型有一定排序能力，但不是特别强。它适合作为教学中的中等强度模型。

注意：AUC 不是钱。AUC 高不等于一定赚钱。商业中还要看触达成本和收益。

## 9. 模型指标 2：Precision

Precision 中文常翻译为精确率。

在发券场景中，它的含义非常直观：

> 我们发券的人里，有多少真的会复购？

假设给 100 人发券，其中 80 人复购：

```text
Precision = 80 / 100 = 80%
```

Precision 高，说明发券命中率高，浪费少。

预算有限时，Precision 非常重要。

## 10. 模型指标 3：Recall

Recall 中文常翻译为召回率。

在复购场景中，它表示：

> 所有真实会复购的人里，我们找到了多少？

假设真实会复购的用户有 1000 人，我们的名单覆盖了 80 人：

```text
Recall = 80 / 1000 = 8%
```

Recall 高，说明覆盖广；Recall 低，说明漏掉很多潜在用户。

预算很少时，我们可能接受低 Recall，因为目标是优先抓住最有价值的一小批人。

## 11. 阈值和 Top N 触达

模型输出的是概率，例如：

| 用户 | 复购概率 |
|---|---:|
| A | 0.91 |
| B | 0.82 |
| C | 0.63 |
| D | 0.21 |

我们必须决定：概率达到多少才触达？

这就是阈值。

阈值高：

- 触达人少
- Precision 通常高
- Recall 通常低
- 适合预算少、追求高命中率

阈值低：

- 触达人多
- Precision 通常下降
- Recall 通常上升
- 适合预算充足、追求覆盖面

在商业中，我们常用 Top N：

```text
触达分数最高的 Top 5%、Top 10%、Top 20% 用户
```

Top N 比固定 0.5 阈值更符合预算管理。

## 12. 经济概念：ROI

ROI 是 Return on Investment，投资回报率。

在本章中，可以简化理解为：

```text
ROI = (预期收益 - 触达成本) / 触达成本
```

假设：

- 触达 1 个用户成本 8 元
- 命中 1 个复购用户带来 80 元毛利

如果触达 100 人，Precision 是 80%，那么预计 80 人复购：

```text
预期收益 = 80 × 80 = 6400 元
触达成本 = 100 × 8 = 800 元
ROI = (6400 - 800) / 800 = 7
```

ROI=7 表示每投入 1 元触达成本，扣除成本后大约带来 7 元回报。

当然，这里是教学简化。真实业务还要考虑优惠券成本、自然复购、毛利差异和对照组。

## 13. 对照组是什么

如果我们给用户发券后，他复购了，能不能说是发券带来的？

不一定。

他可能本来就准备买。发券只是让他少付钱。

所以需要对照组。

对照组就是从相似用户中留出一部分不触达，然后比较触达组和对照组的复购差异。

如果触达组复购率明显高于对照组，才说明触达带来了增量。

## 19. 教师扩展讲稿：为什么复购比“再买一次”更重要

同学们，复购看起来只是“用户又买了一次”，但在商业上它远远不只是一次交易。一个用户愿意再次购买，说明他至少跨过了三道心理门槛。

第一道门槛是信任。第一次购买时，用户可能只是试一试；第二次购买时，他已经用自己的行为表达了“这家店基本可靠”。这份信任会降低企业未来的沟通成本。

第二道门槛是习惯。很多商业模式的价值不在第一次成交，而在用户形成习惯之后的长期购买。超市、外卖、会员电商、内容平台、SaaS 软件都是如此。用户一旦形成习惯，企业就不必每次都用高额广告费重新把他买回来。

第三道门槛是关系。复购用户通常更容易接受新品推荐、会员权益、组合销售和高客单价商品。企业对他的理解越深，后续运营越精细。

所以，复购预测的本质不是“预测一个标签”，而是回答一个资源配置问题：运营资源有限，优惠券有限，客服和短信触达次数有限，我们应该优先把资源给谁？

## 20. 必须讲清楚的业务概念

### 20.1 拉新与留存

拉新是获得新用户，留存是让已有用户留下来并继续产生价值。很多企业早期靠拉新增长，但拉新成本会越来越高。如果老用户留不住，公司就像一个漏水的桶，广告部门不停往桶里倒水，运营部门却没有把漏洞补上。

复购预测服务的是留存与复购。它不是替代拉新，而是帮助企业减少对高成本拉新的依赖。

### 20.2 LTV：客户生命周期价值

LTV 是 Lifetime Value，客户生命周期价值。它表示一个客户在未来一段时间内可能为企业贡献的总价值。

直观理解，如果一个用户第一次买了 50 元商品，但之后每个月都会购买，他的价值可能远高于一个第一次买了 300 元但再也不来的用户。

复购预测与 LTV 密切相关。高复购概率用户通常更可能拥有较高 LTV，但两者不是完全相同。复购概率回答“会不会再买”，LTV 更进一步回答“未来大概值多少钱”。

### 20.3 触达

触达是企业主动联系用户的动作，比如发短信、发优惠券、推送 App 消息、电话回访、站内信提醒等。

触达不是免费的。即使短信成本很低，频繁打扰也会消耗用户耐心。优惠券更有真实成本，因为它会减少利润。所以，复购预测的价值在于让触达更精准。

### 20.4 优惠券补贴

优惠券不是礼物，而是经营工具。它可以促进购买，也可能造成浪费。

如果用户本来就会买，你给他优惠券，企业只是少赚了钱；如果用户不会买，但优惠券促成了购买，才可能产生增量价值。因此，发券策略必须和对照组结合，验证优惠券是否真的带来增量复购。

## 21. 模型指标的通俗解释

AUC 可以理解为模型的排序能力。它不要求我们马上决定发不发券，而是看模型能否把更可能复购的人排在前面。AUC=0.5 接近随机猜，AUC 越接近 1，排序能力越强。

precision 是命中率。假设我们选择模型评分最高的 100 个用户发券，其中 80 个后来复购，那么 precision 就是 80%。运营部门很关心 precision，因为它影响触达资源是否浪费。

recall 是找回率。假设全部会复购的用户有 1,000 个，模型选中的名单覆盖了其中 100 个，那么 recall 是 10%。如果企业目标是尽量不漏掉潜在用户，recall 就重要。

Top N precision 是业务最常用的指标。因为运营部门通常不会触达所有用户，只会触达预算允许的一部分人。模型整体 AUC 不错，但 Top 5% 不准，业务价值仍然有限。

## 22. 课堂案例讲法

可以把班级分成两组。一组代表“广撒网发券”，另一组代表“模型名单发券”。

广撒网方案是给所有用户发 20 元券。优点是简单，缺点是成本高，很多本来会买的人也拿到券。

模型名单方案是只给复购概率最高的 Top 5% 用户发券。优点是预算集中，缺点是可能漏掉一些潜在用户。

然后让学生用本章结果讨论：Top 5% precision 约 80.79%，ROI 约 7.08，这说明模型名单有较强运营价值。但还要提醒学生：这是基于历史数据和模拟策略的估算，不是最终因果证明。真正上线时仍要保留对照组。

这样讲，学生会明白模型不是为了“分数好看”，而是为了让企业少浪费钱、多做有效动作。
